# Contributing to Rust in 90 Days

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Ways to Contribute

1. **Report Bugs** - Found an error? Let us know!
2. **Improve Content** - Better explanations, more examples
3. **Fix Typos** - Even small fixes are appreciated
4. **Add Exercises** - More practice problems help learners
5. **Translate** - Help make this accessible globally

## Content Guidelines

### Lesson Structure

Each lesson should follow this structure:

1. **Goal** (1 min) - Clear, achievable objective
2. **Concept** (3 min) - Core explanation
3. **Code** (4 min) - Practical examples
4. **Takeaways** (1 min) - Key points
5. **Pitfalls** - Common mistakes
6. **Challenge** - Practice exercise

### Writing Style

- **Clear and concise** - 10-minute lessons
- **Beginner-friendly** - Explain jargon
- **Code-focused** - Show, don't just tell
- **Practical** - Real-world examples
- **Encouraging** - Keep it positive

### Code Examples

- Use clear variable names
- Include comments for complex logic
- Show expected output
- Test all code examples
- Keep examples focused and minimal

## Submitting Changes

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test locally with `npm run docs:dev`
5. Submit a pull request

## Pull Request Process

1. Update relevant documentation
2. Follow the existing style
3. Write clear commit messages
4. Test your changes
5. Wait for review

## Questions?

Open an issue or start a discussion!
